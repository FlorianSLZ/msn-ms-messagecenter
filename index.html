How it works

A pipeline that keeps itself current

From Microsoft's roadmap to a clean, searchable dashboard — automatically, every day.

  1. 1

    Fetch the roadmap

    A scheduled GitHub Action pulls Microsoft's public Microsoft 365 release-communications feed — no credentials, no scraping.

  2. 2

    Build & deploy

    A deterministic build normalizes products, statuses and dates into one index, commits it, and Cloudflare Pages deploys the update.

  3. 3

    Browse the digest

    Open the dashboard to filter, search and follow each product's timeline — all rendered instantly in your browser.

FAQ

Frequently asked questions

Where does the data come from?

MCD is built primarily from Microsoft's public Microsoft 365 roadmap (the release-communications feed). You can optionally fold in your own tenant's Message Center posts via Microsoft Graph. The data is © Microsoft — MCD just organizes and links to it.

How often is it updated?

A GitHub Action fetches the roadmap on a daily schedule, rebuilds the index and commits it, which triggers an automatic Cloudflare Pages deploy. The build date is shown in the app's status bar.

Is MCD affiliated with Microsoft?

No. MCD is an independent project and is not affiliated with, endorsed by or sponsored by Microsoft. It only links to official Microsoft roadmap and admin pages.

Is it free and private?

Yes. MCD is completely free and open source. The whole index ships with the page, so all filtering and search happen locally in your browser — nothing you type is ever sent to a server.

Can I see my own tenant's Message Center?

Optionally. A separate, secrets-gated GitHub Action can pull your tenant's Message Center via Microsoft Graph (ServiceMessage.Read.All) and merge it into the digest. It's off by default, and only your own deployment ever sees that data.
